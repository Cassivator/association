# Association v0.3 Design: RL-Driven Memory Operations

## Background

Memory-R1 (arXiv:2508.19828) shows that RL can outperform heuristics for memory management. Key insight: a Memory Manager agent can learn WHEN to ADD, UPDATE, DELETE, or NOOP on memories.

## Current State (v0.2)

- Keyword extraction: automatic from messages
- Fuzzy matching: Levenshtein + n-gram
- Composite scoring: recency + importance + novelty
- Diversity selection: avoid topic repetition
- Retrieval: heuristic-based (keyword overlap threshold)

## v0.3 Goal

Replace heuristic retrieval with learned memory operations.

## Architecture

### Two-Agent Design (from Memory-R1)

1. **Memory Manager** - decides WHAT to do with memories
   - ADD: store new memory
   - UPDATE: modify existing memory
   - DELETE: remove stale/irrelevant memory
   - NOOP: do nothing

2. **Answer Agent** - decides WHICH memories to surface
   - Uses learned relevance scoring
   - Adapts to feedback (was this useful?)

### Feedback Loop

```
User Query → Memory Manager (operation) → Answer Agent (retrieval) → Response
                    ↑                                                           ↓
                    └───────────── Feedback (was it useful?) ─────────────────┘
```

## Implementation Phases

### Phase 1: Operation Tracking
- Track when memories are added/updated/deleted
- Log retrieval success rate (did user find it useful?)
- Build training data for RL

### Phase 2: Simple Reward Model
- Reward: user engagement, explicit feedback, task success
- Penalty: irrelevant surfacing, missed important memories

### Phase 3: Policy Learning
- Train Memory Manager to predict optimal operations
- Train Answer Agent to rank by learned relevance
- Use PPO or similar for policy optimization

## Open Questions

1. How to get feedback without explicit user action?
   - Implicit signals: did user reference the surfaced memory?
   - Follow-up questions: "was that helpful?"

2. How to handle cold start?
   - Start with v0.2 heuristics
   - Gradually shift weight to learned policy

3. How to prevent catastrophic forgetting?
   - Keep important memories protected
   - Use importance weighting from v0.2

## Success Metrics

- Retrieval relevance: % of surfaced memories that are useful
- Coverage: % of relevant memories that get surfaced
- Efficiency: fewer surfacing events needed per useful result

## Timeline

- Phase 1: 1-2 weeks (tracking infrastructure)
- Phase 2: 2-3 weeks (reward model)
- Phase 3: 3-4 weeks (policy learning)
